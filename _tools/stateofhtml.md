## What it does

An annual survey measuring developer awareness and usage of HTML features, browser APIs, and accessibility primitives. Goes beyond markup to cover built-in browser capabilities — form controls, media elements, web components, and emerging features like the Popover API or `<dialog>` — and collects sentiment on what's missing or painful to use.

## Who is it for

Web developers tracking which HTML features are safe to use, browser vendors and standards bodies prioritising implementation work, and accessibility practitioners understanding where native HTML still falls short of developer needs.

## Where to find it

[stateofhtml.com](https://stateofhtml.com) — results are published annually, with each year's data accessible at a versioned URL (e.g. 2023.stateofhtml.com).

## Who is maintaining it

Devographics, with Lea Verou leading the survey design for HTML. The survey tooling is open source at [github.com/Devographics](https://github.com/Devographics).
